  ИНСТРУКЦИЯ ПО ЗАПУСКУ: AES-256-GCM Encryption Tool
═══════════════════════════════════════════════════════════

📋 ШАГ 1: ПРОВЕРКА NODE.JS
──────────────────────────────────────────────────────────
Откройте терминал (командную строку) и выполните:

  node --version

Если Node.js не установлен:
• Windows/Mac: скачайте с https://nodejs.org (выберите LTS версию)
• Linux: выполните команду
  sudo apt install nodejs npm


📋 ШАГ 2: СОЗДАНИЕ ПРОЕКТА
──────────────────────────────────────────────────────────
1. Создайте папку для проекта:
   mkdir aes-tool
   cd aes-tool

2. Сохраните код в файл encrypt.js

3. Инициализируйте проект:
   npm init -y


📋 ШАГ 3: УСТАНОВКА ЗАВИСИМОСТЕЙ (опционально)
──────────────────────────────────────────────────────────
Для работы с XLSX файлами выполните:

  npm install xlsx

⚠️  БЕЗ этой библиотеки работают только TXT и CSV файлы!


📋 ШАГ 4: ЗАПУСК ПРОГРАММЫ
──────────────────────────────────────────────────────────
В терминале выполните:

  node encrypt.js

Программа запустится в интерактивном режиме.


📋 ШАГ 5: РАБОТА С ПРОГРАММОЙ
──────────────────────────────────────────────────────────

🔹 ВЫБОР РЕЖИМА:
   1 - Шифрование (encrypt)
   2 - Дешифрование (decrypt)

🔹 ВВОД ПАРОЛЯ:
   Введите пароль (минимум 16 символов для безопасности)
   ⚠️  ЗАПОМНИТЕ ПАРОЛЬ! Без него данные не восстановить!

🔹 ВЫБОР ИСТОЧНИКА ДАННЫХ:
   
   [1] Ручной ввод
       - Введите текст вручную
       - Результат отобразится в консоли
   
   [2] Файл без заголовков
       - Поддержка: .txt, .csv, .xlsx
       - Обрабатывается первый столбец
       - Пример: input.txt
       - Результат: input_encrypt.txt
   
   [3] Файл с одним заголовком
       - Поддержка: .csv, .xlsx
       - Укажите название столбца (например: password)
       - Результат: data_encrypt.csv
   
   [4] Файл с несколькими заголовками
       - Поддержка: .csv, .xlsx
       - Введите столбцы через запятую: email,password,token
       - Результат: users_encrypt.xlsx


📋 ПРИМЕРЫ ЗАПУСКА
──────────────────────────────────────────────────────────

ПРИМЕР 1: Зашифровать текстовый файл
────────────────────────────────────
  $ node encrypt.js
  
  Режим: 1
  Пароль: MySecurePassword123
  Источник: 2
  Путь: secrets.txt
  
  → Результат: secrets_encrypt.txt


ПРИМЕР 2: Расшифровать CSV с паролями
──────────────────────────────────────
  $ node encrypt.js
  
  Режим: 2
  Пароль: MySecurePassword123
  Источник: 3
  Путь: encrypted.csv
  Столбец: encrypted_password
  
  → Результат: encrypted_decrypt.csv


ПРИМЕР 3: Зашифровать несколько столбцов
─────────────────────────────────────────
  $ node encrypt.js
  
  Режим: 1
  Пароль: MySecurePassword123
  Источник: 4
  Путь: users.xlsx
  Столбцы: email,password,api_key
  
  → Результат: users_encrypt.xlsx


📋 СТРУКТУРА ФАЙЛОВ
──────────────────────────────────────────────────────────
После установки у вас будет:

  aes-tool/
  ├── encrypt.js          ← Основной файл программы
  ├── package.json        ← Метаданные проекта
  ├── node_modules/       ← Библиотеки (если установлена xlsx)
  ├── secrets.txt         ← Ваши входные файлы
  └── secrets_encrypt.txt ← Результаты работы


📋 ЧАСТЫЕ ПРОБЛЕМЫ
──────────────────────────────────────────────────────────

❌ "node: command not found"
   → Установите Node.js с https://nodejs.org

❌ "Библиотека xlsx не установлена"
   → Выполните: npm install xlsx

❌ "Файл не найден"
   → Укажите полный путь к файлу
   → Или поместите файл в папку с encrypt.js

❌ "Ошибка аутентификации"
   → Проверьте правильность пароля
   → Убедитесь, что данные не повреждены

❌ "Столбец не найден"
   → Проверьте название столбца (с учётом регистра)
   → Посмотрите первую строку CSV/XLSX файла

──────────────────────────────────────────────────────────
Если Node.js уже установлен:

  1. mkdir aes-tool && cd aes-tool
  2. [сохраните код в encrypt.js]

📋 ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ
──────────────────────────────────────────────────────────
• Алгоритм: AES-256-GCM
• Производный ключ: PBKDF2, 600,000 итераций
• Salt: 32 байта
• IV: 12 байт
• Auth Tag: 16 байт
• Кодирование: Base64
